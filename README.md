# sp500-autoresearch

Karpathy's autoresearch loop adapted for SP500 equities. An AI agent iterates
on `agent.py`, runs backtests, and commits improvements — while you sleep.

**Universe**: ~500 SP500 stocks, daily bars, 11 years (2014–present)  
**Execution**: Signal at close[T] → filled at open[T+1] (no lookahead)  
**Direction**: Long and short allowed  
**Macro**: VIX + Fear & Greed Index included as signals  
**Hardware**: CPU only, Proxmox VM (8 cores / 24 GB RAM)  
**Schedule**: Daily cron at 16:30 ET → agent runs experiments until midnight  

---

## Baseline Performance (pre-established, do not re-test)

RSL / Jegadeesh-Titman momentum, ~400 stocks, $100k capital, 2015–2026:

| Config | Return | Sharpe | MaxDD | Final |
|--------|--------|--------|-------|-------|
| **JT skip=4 + top3% + inv-vol + MA20 + stop20%** | **+4,455%** | **1.37** | **-33.7%** | **$4.6M** |
| skip=4 + MA filter only | +3,780% | 1.35 | -32.5% | $3.9M |
| skip=4 + stop20% only | +3,016% | 1.26 | -27.7% | $3.1M |
| skip=4 raw | +2,471% | 1.21 | -28.8% | $2.6M |
| skip=0 (no JT) | +1,202% | 1.00 | -30.1% | $1.3M |

The agent's job is to beat **Sharpe 1.37** on the held-out validation set (last 2 years).
See `program.md` for full findings and what to explore next.

## File Structure

```
prepare.py        — FROZEN. Data loader, backtest engine, metrics, 60-min timeout.
agent.py          — AGENT EDITS THIS. Strategy, signals, METRIC, HYPOTHESIS.
run.py            — Thin wrapper: loads data → generates signals → prints results.
refresh_data.py   — Downloads SP500 OHLCV + VIX + Fear & Greed via yfinance.
log_exp.sh        — AGENT CALLS THIS after every run. Appends result row to program.md.
program.md        — Research directions, confirmed findings, experiment log.
daily_run.sh      — Cron entry point: refresh data → launch Claude Code in tmux.
setup.sh          — One-shot VM setup: deps, cron, git init.
pyproject.toml    — Python deps (uv).
data/             — Parquet files (gitignored). Generated by refresh_data.py.
logs/             — Session logs (gitignored).
```

## Execution Model

```
close[T]   →   generate_signals()   →   weight[T]
open[T+1]  →   entry fill                          ← realistic
open[T+2]  →   exit / rebalance
P&L[T+1]   =   weight[T] * (open[T+2] / open[T+1] - 1)
```

No lookahead: `generate_signals()` never sees `open[T+1]` when deciding at `close[T]`.  
Costs: 5 bps per unit of turnover, enforced by `prepare.py`.

## 7. Kick Off the Loop

Cron fires automatically at 16:30 ET. For a manual first run:

```bash
./daily_run.sh
tmux attach -t sp500
```

Then paste this prompt into Claude Code if needed:

```
Read program.md carefully. Find the next experiment number in the log.
Form a hypothesis, edit agent.py (set HYPOTHESIS, METRIC, generate_signals),
run `uv run python run.py`, then IMMEDIATELY run:
  ./log_exp.sh NNN "hypothesis" "sharpe=X.XX calmar=X.XX alpha=X.XX maxdd=-XX.X% ret=+XXX%" Yes|No
using the exact numbers from run.py output. This is MANDATORY before any new code.
Commit with message "exp#NNN: <hypothesis> | <metric>=<score>" if improved, revert if not.
Immediately start the next experiment. Stop at 23:55.
```

## Research Frontier

The agent explores these in priority order (see `program.md` for full details):

1. **Short side** — short bottom 3% momentum, VIX-gated
2. **VIX regime filter** — reduce/exit longs when VIX > 25
3. **Fear & Greed contrarian** — extreme readings as position-size modifier
4. **Momentum + short-term reversal hybrid** — different holding periods
5. **Volume breakout confirmation** — filter weak momentum signals
6. **52-week high proximity** — George & Hwang factor overlay
7. **ML signal layer** — LightGBM on rolling features, trained on first 9 years
8. **Turtle short side** — 20-day breakdown entries, ATR sizing

## Tips

- Check `git log --oneline` in the morning to see what the agent committed
- Edit `program.md` to nudge research direction at any time
- Multiple VMs = multiple parallel agents on separate git branches
- The 60-min timeout is in `prepare.py` — lower to 15 min for rule-based-only runs
- Data is gitignored — re-run `refresh_data.py` on a fresh clone
